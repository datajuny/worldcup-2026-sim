# 2026 월드컵 몬테카를로 시뮬레이션

2026 북중미 월드컵(48개국, 12개 조, 총 104경기)을 시드 고정 몬테카를로 10,000회로 시뮬레이션하고, 그 결과(우승 확률·라운드별 진출 확률·한국 전망·다크호스)를 보여주는 순수 정적 사이트입니다. 서버나 빌드 도구 없이 HTML 파일 하나와 JSON 데이터로 동작합니다.

시뮬레이션 기준일은 2026-06-07이며, 우승 확률 상위는 프랑스 28.61%, 스페인 13.00%, 아르헨티나 12.06%입니다. 한국은 A조(멕시코·남아프리카공화국·체코)에서 32강 진출 확률 79.7%로 계산됐습니다. 이 수치를 어디까지 믿어도 되는지는 아래 "이 결과를 실제 예측으로 믿으면 안 되는 이유"를 꼭 읽어주세요.

## 로컬 실행

정적 사이트라 두 가지 방법 중 아무거나 됩니다.

```bash
# 방법 1. 그냥 열기
# index.html을 더블클릭 (파일 프로토콜로 열림)

# 방법 2. 로컬 서버 (fetch 차단 등 브라우저 보안 정책에 안전)
python3 -m http.server
# → http://localhost:8000 접속
```

## 시뮬레이션 재실행

```bash
python3 scripts/simulate.py
```

- **표준 라이브러리만 사용합니다** (json, math, os, random, time, itertools). pip 설치가 필요 없습니다.
- 시드 42 고정, 10,000회 반복(회당 104경기)이라 누가 언제 돌려도 같은 결과가 나옵니다.
- 출력은 `data/worldcup-2026.json`에 덮어써집니다. 입력 데이터(랭킹·폼·시장가치·부상)는 스크립트 상단에 조사 결과 그대로 들어 있습니다.

## GitHub Pages 배포

1. GitHub에서 새 레포를 만들고 이 폴더를 푸시합니다.
   ```bash
   git init
   git add .
   git commit -m "worldcup 2026 monte carlo sim"
   git remote add origin https://github.com/{계정}/{레포}.git
   git push -u origin main
   ```
2. 레포의 **Settings → Pages**로 이동합니다.
3. Source를 **Deploy from a branch**로, Branch를 **main / (root)**로 설정하고 저장합니다.
4. 잠시 후 `https://{계정}.github.io/{레포}/`에서 사이트가 열립니다.

**`.nojekyll`을 포함한 이유**. GitHub Pages는 기본적으로 Jekyll 빌드를 거치는데, 이 과정에서 언더스코어(`_`)로 시작하는 파일·폴더가 무시되거나 불필요한 빌드 단계가 끼어듭니다. 루트에 빈 `.nojekyll` 파일이 있으면 Jekyll을 건너뛰고 파일을 올린 그대로 서빙합니다. 이 프로젝트는 순수 정적 파일이라 Jekyll이 필요 없습니다.

## 모델 가정과 가중치

팀 전력 점수 산식은 `data/worldcup-2026.json`의 `model.strengthFormula` 그대로입니다.

```
S = 0.55·norm(fifaPoints) + 0.20·(최근 10경기 승점률) + 0.25·norm(log(시장가치 MEUR))
    − 부상 페널티 + 커버리지 보정상수(상세 데이터 8개국에만 적용)
```

| 가중치 | 값 | 근거 (JSON model.weights 원문) |
|---|---|---|
| rank | 0.55 | FIFA 포인트는 수년치 공식 경기 결과를 누적 반영하는 가장 안정적인 단일 지표라 베이스로 최대 가중 |
| form | 0.20 | 최근 10경기는 현재 컨디션을 반영하지만 표본이 작고 친선전이 섞여 노이즈가 커서 보조 가중 |
| value | 0.25 | 스쿼드 시장가치(로그 스케일)는 랭킹이 못 잡는 선수단 체급 차이의 대리 지표 — 랭킹 포인트가 거의 같은 1~3위(프랑스·스페인·아르헨티나)를 변별하는 핵심 항목 |
| injuryPenalty | 0.02 | 확인된 주전 부상 보도가 있는 잉글랜드(Livramento·Colwill 제외, James·Spence 복귀 직후, Stones 피트니스 우려)에만 0.02 감점 — 타 팀은 부상 데이터 미확보라 0 |
| coverageBiasCorrection | 0.1176 | 상세 데이터 8개국의 평균 점수를 같은 8개국의 랭킹 단독 점수 평균과 일치시키는 상수 — 데이터가 있다는 이유로 코호트 전체가 랭킹 단독 40개국 대비 불리해지는 비대칭 제거 (코호트 내부 서열은 폼·시장가치·부상으로 결정) |

경기 결과는 양 팀 독립 Poisson으로 뽑습니다 (`model.matchModel` 그대로).

```
lambda_A = 1.22 × exp(1.15 × (S_A − S_B))
동률 시 연장(λ×1/3 Poisson) → 승부차기(전력차 반영 0.5±0.15)
```

baseGoals 1.22와 alpha 1.15는 경기당 평균 총득점이 최근 월드컵 실측대(2.64~2.69골)에 들어오도록 캘리브레이션한 값이며, 이번 실행의 실제 평균은 2.69골입니다(정규 90분 득점 기준, 104경기 전체 평균).

## ★ 이 결과를 실제 예측으로 믿으면 안 되는 이유

JSON의 `model.limitations`에 기록된 한계를 요약하면 이렇습니다.

**데이터 비대칭이 큽니다.**
- 폼·시장가치·부상은 FIFA 상위 8개국만 조사됐고, 나머지 40개국은 FIFA 랭킹 100%로만 계산됩니다. 9~16위권(독일·벨기에·크로아티아 등)의 폼·시장가치가 미반영이라 이들 점수는 상대적으로 보수적입니다.
- 스페인·잉글랜드 시장가치는 EUR 확정치가 없어(USD 2차 보도만) 상세 코호트 평균 norm 값(0.5212)으로 대체(imputed)했습니다.
- 부상 페널티가 잉글랜드에만 있는 건 다른 팀이 건강해서가 아니라, 부상 데이터가 잉글랜드만 확보됐기 때문입니다. "페널티 0 = 데이터 없음"입니다.

**표본이 작고 노이즈가 섞인 입력이 있습니다.**
- 폼은 최근 10경기뿐이고 친선전이 섞여 있습니다.
- 프랑스의 코트디부아르전은 소스끼리 결과가 상충해(ESPN 1-2 패 vs 타 소스 0-0 무) ESPN 표기를 채택했고, 모로코 폼에는 AFCON 결승 몰수 3-0 승 판정이 승으로 들어가 있습니다.

**대회 규칙 일부는 근사입니다.**
- 팀 컨덕트 스코어(옐로·레드카드)는 데이터가 없어 조 순위·3위 비교에서 모델링하지 못했고, 조 내 동률 시 헤드투헤드 규정도 미적용입니다.
- 3위 8팀의 32강 슬롯 배정은 FIFA 규정 Annex C의 495가지 조합표 대신 이분 매칭 근사를 썼습니다(폴백 발생 0/10000회).
- 16강 이후 대진은 조사 데이터에 없어 32강 승자를 번호 순서대로 페어링하는 근사라, 실제 FIFA 브래킷과 다를 수 있습니다.

**모델에 아예 없는 변수가 많습니다.**
- 감독 전술, 선수 컨디션·피로, 날씨·기온, 이동 거리, 홈 어드밴티지(개최 3국 미국·캐나다·멕시코 보정 없음), 승부차기 전문 능력은 전부 미반영입니다.

결국 이 결과는 단정 예측이 아니라 "이 가정과 이 데이터 아래에서" 시드 고정 몬테카를로가 내놓은 확률 분포입니다. 모델 가정이 바뀌면 수치도 바뀝니다.

## 이 워크플로우를 다른 주제에 재사용하기

선거 의석 분포, 다른 스포츠 토너먼트, 시장 점유율 경쟁 같은 "여러 주체가 경쟁하는 불확실한 결과"에 같은 구조를 그대로 쓸 수 있습니다. 먼저 리서치 결과를 `sources.md`에 출처·신뢰도·상충 처리까지 기록하고, 도메인의 핵심 지표 2~4개를 정규화해 가중 합성한 단일 강도 점수를 만듭니다. 그 점수 차이를 도메인에 맞는 확률 분포(축구는 Poisson, 선거는 정규분포 스윙, 점유율은 디리클레 등)에 넣고 시드 고정 몬테카를로를 돌리면 재현 가능한 확률 분포가 나옵니다. 데이터가 일부 주체에만 있는 비대칭은 커버리지 보정상수로 중립화하고, 가중치 근거와 한계(limitations)를 결과 JSON 안에 같이 박아두면 나중에 누가 봐도 "왜 이 수치인지"를 추적할 수 있습니다. 마지막으로 결과 JSON을 정적 HTML 하나로 시각화하면 GitHub Pages에 그대로 배포됩니다.

## 파일 구조

```
worldcup-2026-sim/
├── index.html              # 결과 시각화 (정적, data/worldcup-2026.json 사용)
├── data/worldcup-2026.json # 시뮬레이션 결과 + meta/model/limitations
├── scripts/simulate.py     # 몬테카를로 시뮬레이터 (표준 라이브러리만)
├── sources.md              # 데이터 출처 26개 + 미확보 데이터 + 상충 처리
├── .nojekyll               # GitHub Pages Jekyll 빌드 비활성화
└── README.md
```
